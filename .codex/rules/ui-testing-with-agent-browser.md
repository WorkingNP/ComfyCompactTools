# UI Testing with Agent Browser

## 原則
UI変更時は、手動確認の前に**Agent Browser**で自動確認すること。

## 使用タイミング
- UIコンポーネント実装・修正時
- レイアウト変更時
- フォーム動作確認時
- 画像ギャラリー等の表示確認時

## 実行方法
1. `e2e-runner`エージェントを起動（Claude Codeが自動起動）
2. テストシナリオ生成（例：ワークフロー選択→パラメータ入力→生成）
3. スクリーンショット・動画・トレース取得
4. 自動でアーティファクトアップロード

## 手動確認が必要なケース
- 色・フォント等の微細なデザイン確認
- アニメーション動作
- Agent Browserでテスト失敗した場合の原因調査

## テスト失敗時の対応
1. トレースファイル（.zip）を確認
2. スクリーンショットで視覚的問題を特定
3. 動画で操作フローを再生
4. 必要なら手動でブラウザ確認
