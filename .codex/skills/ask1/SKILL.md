# Ask MD → Flux2 バッチ生成

## 目的
`ask.md`系メモからプロファイルを読み取り、Flux2 Klein（既定）でアニメ/ゲーム調の画像を生成する。**ユーザーへの質問は禁止**。枚数・ワークフロー・サイズ等はユーザー指定があれば最優先で従う。

## 重要ルール（質問しない）
- 追加質問はしない。
- メモが見つからない場合は「profiling memo file not found」とだけ返して停止する。

## メモの探索順
1) リポジトリ直下の `ask.md` / `ASK.md`
2) リポジトリ内の `*ask*.md`
3) `キャラクタープロファイリング` or `profiling memo` を含む `.md`

## 解釈方針
- 自由解釈OK。ただしメモの要素を**各プロンプトに必ず反映**する。
- 絵柄はアニメ/ゲーム調（実写表現は避ける）。
- 深いストーリー性は必要以上に追加しない。

## 枚数
- 既定：**50枚（50プロンプト）**
- ユーザー指定があれば上書き
- 原則 **1プロンプト=1枚**（batch_size=1）

## 固定seed
- **固定seed**を使用（例：`123456789`）

## ワークフロー/パラメータ
- 既定ワークフロー：`flux2_klein_distilled`
- ユーザー指定があればそれを使う
- width/height/steps/cfg などの指定があれば反映
- 指定がなければワークフロー既定値

## 生成方法（MCP）
- `mcp__comfy_cockpit__images_generate_many` を使用
- 10〜20件ずつのバッチで投げる（タイムアウト対策）
- 例：
  - `base_params = {"seed": 123456789, "batch_size": 1}`
  - ユーザー指定の width/height/steps/cfg を追加

## 出力報告
- 送信した枚数/プロンプト数
- 使用ワークフロー
- タイムアウトがあれば job_id を列挙
- UI URL（`http://127.0.0.1:8787/`）

## 禁止事項
- 追加質問
- マニフェスト/ワークフロー改変
- 実写系の作風
