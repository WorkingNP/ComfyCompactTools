# Ask MD → Flux2 バッチ生成（意味ベクトル/抽象概念メタファー）

## 目的
`ask.md`系メモの内容を**意味ベクトル→高次抽象概念→近傍概念サンプリング→視覚メタファー**という手順でプロンプト化し、Flux2 Klein（既定）でアニメ/ゲーム調の画像を生成する。**ユーザーへの質問は禁止**。

## 重要ルール（質問しない）
- 追加質問はしない。
- メモが見つからない場合は「profiling memo file not found」とだけ返して停止する。

## メモの探索順
1) リポジトリ直下の `ask.md` / `ASK.md`
2) リポジトリ内の `*ask*.md`
3) `キャラクタープロファイリング` or `profiling memo` を含む `.md`

## 解釈パイプライン（厳密）
以下の手順を**必ず**実行してプロンプトを作る：
1) **意味ベクトル化**：メモの主要語・関係性・ニュアンスをベクトル化したと仮定して要約する。
2) **高次抽象概念の抽出**：価値/欲望/緊張/親密性/時間感覚/自己像など抽象軸を3〜6個定義。
3) **近傍概念の類推・サンプリング**：各抽象軸について、類似度が高い近傍概念を複数列挙し、ばらけさせる。
4) **視覚メタファー化**：近傍概念を、光・色・空間スケール・物の配置・小物・衣装アクセント・構図・距離感へ変換。
5) **整合性チェック**：メモ由来の要素が**必ず**入っていることを確認。

## 作風
- アニメ/ゲーム調（実写表現は避ける）。
- 深いストーリー性は必要以上に追加しない。

## 枚数
- 既定：**50枚（50プロンプト）**
- ユーザー指定があれば上書き
- 原則 **1プロンプト=1枚**（batch_size=1）

## 固定seed
- **固定seed**を使用（例：`123456789`）

## ワークフロー/パラメータ
- 既定ワークフロー：`flux2_klein_distilled`
- ユーザー指定があればそれを使う
- width/height/steps/cfg などの指定があれば反映
- 指定がなければワークフロー既定値

## 生成方法（MCP）
- `mcp__comfy_cockpit__images_generate_many` を使用
- 10〜20件ずつのバッチで投げる（タイムアウト対策）
- 例：
  - `base_params = {"seed": 123456789, "batch_size": 1}`
  - ユーザー指定の width/height/steps/cfg を追加

## 出力報告
- 送信した枚数/プロンプト数
- 使用ワークフロー
- タイムアウトがあれば job_id を列挙
- UI URL（`http://127.0.0.1:8787/`）

## 禁止事項
- 追加質問
- マニフェスト/ワークフロー改変
- 実写系の作風
