# ux-ヒアリング 成功例：画像拡大機能

## 対話ログ

**AI**: 何を実現したいですか？

**ユーザー**: 画像を拡大したい

**AI**: なぜ拡大したいですか？どんな時に使いますか？

**ユーザー**: 細かい部分を確認したい。顔の表情とか、テクスチャとか

**AI**: なるほど。拡大して成功したら、どうなっていますか？

**ユーザー**: 見たい部分がはっきり見える。ぼやけてない

**AI**: 逆に、失敗したらどうなりますか？

**ユーザー**: 拡大しすぎて画像が見切れる。スクロールしないと全体が見えなくなる

**AI**: 操作の流れを確認させてください。どこをクリックして、何が起きますか？

**ユーザー**: 画像をクリックしたら拡大される。もう一回クリックで元に戻る

---

## 出力

```markdown
## User Journey: 画像拡大

### ユーザーの意図
- 画像の細部（顔の表情、テクスチャ）を確認したい
- 拡大後もスクロールなしで全体を把握したい

### 操作の流れ
1. 画像が表示されている
2. 画像をクリックする
3. 画像が拡大される（見たい部分がはっきり見える）
4. もう一度クリックすると元に戻る

### 成功条件
- 拡大後の画像がぼやけていない
- 拡大後の画像が画面内に収まっている（スクロール不要）

### 失敗条件
- 画像が見切れる（スクロールが必要になる）
- 拡大しても画質がぼやける
```

---

## なぜ成功か
- 「拡大したい」という表面的な要望から、「細部を確認したい」という本質的な意図を引き出した
- 「見切れる」という失敗条件を明確化し、テスト可能な形に落とし込んだ
- 操作の流れが具体的で、E2Eテストに直結する
