resumeで再開ってどういう意味？停止状態があるの？

回答:
- いったんTUIを終了した「過去の対話セッション」を、履歴から選んで続きを開く機能です（デフォルトはピッカー、`--last`で直近）。
- いわゆる“プロセスが一時停止して再開”というより、「保存された会話ログ/状態（どのリポジトリで作業していたか等）を読み込んで続ける」イメージです。

forkって分岐前のコンテキストを保持しているってこと？

回答:
- はい。過去セッションをベースに「別セッションとしてコピーして続き」を作る機能です（会話コンテキストを引き継ぎつつ、以降のやり取りは別々に分岐）。
- “同じ地点から別案を試したい/リスク高い変更を別線でやりたい”ときに使います。

execは純粋に意味がわからない。いつどうやってつかう？

回答:
- `exec` は「TUIを立ち上げずに」Codexを1回走らせる非対話モードです（自動化/スクリプト/CI向け）。
- 使い方例:
  - `codex exec "このリポジトリでLintを直して"` のように1発で依頼して終了させる
  - `echo "指示" | codex exec -` のようにstdinから指示を渡す
  - ログが欲しいなら `--json`（JSONLイベント出力）や `--output-last-message <file>`
- `codex exec resume` は、過去の `exec` 実行（セッション）をIDや `--last` で続きを走らせる用途です。

cloudは、すでにcloudでやらせたタスクの進捗を見るの？それともcloud上でやらせたいとき？

回答:
- 両方です。`codex cloud` は Codex Cloud のタスクを「一覧/状況確認」も「新規投入」もできます。
- 具体例:
  - 新規に投げる: `codex cloud exec ...`
  - 既存タスクを見る: `codex cloud list` / `codex cloud status`
  - ローカルに反映: `codex cloud diff` / `codex cloud apply`

codexのmcp-serverとして起動すれば、claude codeなどで呼べるって意味だよね？

回答:
- だいたいその理解でOKです。`codex mcp-server` は Codex を MCP サーバとして起動（stdio transport）し、MCPクライアント側（例: Claude Code等）から“ツール”として呼び出せるようにするものです。
- 実際には、クライアント側の設定で「MCPサーバとして `codex mcp-server` を起動する」ように登録する必要があります。

/newは分岐しない新しいスレッドターミナル開く？

回答:
- 「分岐（fork）ではなく」まっさらな新規スレッド/セッションを開始する操作です。
- 新しい“ターミナルウィンドウ”を開くというより、同じTUI内で会話スレッドを切り替えるイメージです。

/diffっていつ使うの？

回答:
- Codexが作業した（または提案している）変更内容を差分で確認したいときに使います。
- 「適用/コミット/次の指示」の前に、何が変わったかをサッとレビューする用途です。

/initって、新しいagents作成？それとも、既存に追記？

回答:
- リポジトリにエージェント向けの指示ファイル（代表例: `AGENTS.md`）の“ひな形”を用意するための初期化コマンドです。
- すでに `AGENTS.md` がある場合に上書きするかどうか等は実行時の挙動/確認に従います（不用意に追記・破壊する用途ではありません）。

モデルの思考時間を変更するには？vscode拡張機能にあるようなやつ

回答:
- `~/.codex/config.toml` の `model_reasoning_effort` を変えます（例: `"low"|"medium"|"high"` のような値）。
- 1回だけ変えたいなら起動時に `-c model_reasoning_effort="low"` のように上書き指定できます。
- 注意: モデル側がそのパラメータを解釈できる場合のみ有効です。

僕のopenai gptプランと連携出来てる？

回答:
- Codex CLI は「OpenAI Platform 側のログイン/課金（組織・プロジェクト）」で動くのが基本で、ChatGPT（いわゆるGPTプラン）のサブスクとは別扱いになることが多いです。
- 同じアカウントでログインしていても、“どの請求先（org/project）で使っているか”はPlatform側の設定に依存します。
